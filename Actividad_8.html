Introducción

Windows Server es una línea de productos para servidores desarrollada por Microsoft Corporation. Windows Server es un sistema operativo de servidor que permite a un equipo manejar funciones de red como servidor de impresión, controlador de dominio, servidor web y servidor de archivos

Instalación en Windows 10

Una vez que descarguemos la imagen ISO y tengamos nuestra USB booteable, colocamos las USB y prendemos el equipo. Lo primero que tenemos que hacer es elegir el idioma y distribución de teclado, una vez que decidamos le damos en siguiente.

En esta parte solo tenemos que presionar el boton de instalar ahora.

Una vez que le damos en instalar ahora la pantalla se pondrá azúl en lo que se inicia el programa de instalación, esto puede demorar varios minutos.

Una vez que cargue el programa de instalación, tendremos que elegir el tipo de sistema operativo que deseamos instalar, en este caso seleccionamos la opción estandar gráfica.

Posteriormente aceptamos los términos y condiciones.

Una vez aceptados los términos tenemos que elegir el tipo de instalación en este caso seleccionamos la opción personalizada.

Seleccionamos el disco duro o partición donde se realizará la instalación.

Una vez seleccionado el disco comenzará la instalación, esto puede tardar algunos minutos o incluso horas.

Cuando la instalación termine el equipo se reiniciará momento en el cual retiraremos la USB y esperamos a que se complete el reinicio.

Esperamos a que se prepare el dispositivo, en mi caso tardo alrededor de 40 minutos.

Una vez que el dispositivo este preparado tendremos que crear al "administrador" proporcionando una contraseña segura.

Enseguida se terminará de configurar el sistema.

Ahora podremos iniciar sesión con los datos del administrador antes creados.

Seleccionamos nuestra red y damos que si a la opción para que otros dispositivos puedan detectar nuestra PC.

Y con esto damos por finalizada la instalación de Windows Server.

 
 

Active Directory

Active Directory (AD) es un servicio de directorio para su uso en un entorno Windows Server. Se trata de una estructura de base de datos distribuida y jerárquica que comparte información de infraestructura para localizar, proteger, administrar y organizar los recursos del equipo y de la red, como archivos, usuarios, grupos, periféricos y dispositivos de red.

Active Directory es el servicio de directorio propietario de Microsoft para su uso en redes de dominio de Windows. Cuenta con funciones de autenticación y autorización y proporciona un framework para otros servicios similares. Básicamente, el directorio consiste en una base de datos LDAP que contiene objetos en red. Active Directory utiliza el sistema operativo Windows Server.

Cuando se habla sobre Active Directory, por lo general nos referimos a los servicios de dominio (Domain Services) de Active Directory, que proporcionan servicios integrados de autenticación y autorización a gran escala.

Características

Esta tecnología permite gestionar los recursos de una Red, tratando estos como objetos, con sus respectivos atributos, para esta gestión, la información de los objetos se guarda en una especie de base de datos de forma jerárquica.

La forma de agruparlos es en árboles, que interactúan entre ellos para su funcionamiento, dependiendo en los dominios donde los recursos estén ubicados, dependiendo de ciertas relaciones de confianza; lo cual permite relaciones de objetos que estén en árboles distintos.

Un directorio es una estructura jerárquica que almacena información acerca de los objetos existentes en la red. Un servicio de directorio, como Active Directory®, proporciona métodos para almacenar los datos del directorio y ponerlos a disposición de los administradores y los usuarios de la red. Por ejemplo, Active Directory almacena información acerca de las cuentas de usuario (nombres, contraseñas, números de teléfono, etc.) y permite que otros usuarios autorizados de la misma red tengan acceso a esa información.

Ejemplos

A la larga toda la funcionalidad de los directorios activos, es la de manejar el acceso a los diferentes dominios (árboles, bosques), por elementos de dominios diferentes, usando los distintos niveles de confianza, según los permisos que se requieran, según la situación.

Bidireccionales: Cuando un domino crea a otro, hay una relación padre-hijo, donde la confianza para el acceso es mutua.

Transitivas: Un dominio padre confía en todos los dominios hijos de sus hijos, y viceversa, lo que da una confianza bidireccional a todos los niveles.

De acceso directo: Si para la comunicación de dos dominios, los mensajes deben pasar chequeos de confianza por los dominios intermedios de la ruta, esta relación de confianza hace que se omitan los nodos intermedios en las cuestiones de comprobación, haciendo más rápida la comunicación entre los dominós remotos.

Relación de confianza externa: son relaciones unidireccionales entre árboles distintos de dominios.

Al crear los dominios, las relaciones bidireccionales y transitivas están implícitas, y se aplican automáticamente, mientras que las relaciones externas y de acceso directo deben configurarse manualmente.
